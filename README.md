This project focuses on UI design, component structure, and frontend data handling.

🧾 Project Overview

This project simulates a personal finance dashboard where users can:

Track income and expenses
View financial summaries
Analyze spending patterns
Interact with transaction data

The application is built purely on the frontend using React (via CDN) and CSS, without any backend dependency.

🎯 Objective

The goal of this project is to demonstrate:

Frontend development skills
UI/UX design thinking
Component-based architecture
State management and interactivity
🧩 Features
📌 1. Dashboard Overview
Total Balance 💰
Total Income 📈
Total Expenses 📉
Savings Rate 🏦
Line Chart → Balance trend over time
Donut Chart → Spending breakdown by category
Bar Chart → Monthly income vs expenses

View all transactions with:
Date
Category
Amount
Type (Income / Expense)
Functionalities:
🔍 Search (by category or amount)
🎯 Filter (income / expense)
🔃 Sorting (date & amount)

📌 3. Role-Based UI (Simulated)
Viewer
Can only view data
Admin
Can add transactions
Can edit transactions

👉 Role switching is handled via UI toggle (no backend)

📌 4. Insights Section

Provides useful financial insights:

Highest spending category
Month-over-month expense comparison
Largest transaction
Expense-to-income ratio
Most active category
📌 5. State Management
Managed using React Hooks (useState, useMemo)
Handles:
Transactions data
Filters & sorting
User role
UI state
